# Characters README

本目录存放角色模板，不存放具体干员的后天成长。

## 目录职责

`characters/` 负责：
- 角色人设
- 角色语言风格
- 角色资源绑定

`characters/` 不负责：
- 某个干员自己的灵魂发展
- 某个端或实例自己的长期偏好
- 项目进行中的私有工作记忆

这些后天内容应进入 `.naga` 或等价的实例存储。

## 当前角色结构

以 `characters/娜杰日达/` 为例：

- `conversation_style_prompt.txt`
  角色人格模板，属于基础系统提示层的一部分
- `娜杰日达.json`
  角色元数据，定义 prompt 文件、头像、Live2D、语音等绑定
- 其余模型、图片、音频资源
  属于角色资产，不属于提示词本身

## 使用原则

1. 修改 `conversation_style_prompt.txt` 等于修改“这个角色的先天模板”。
2. 如果只是某个干员后天学到的东西，不要写回 `characters/`。
3. 同一角色模板可以被多个引擎、多个端、多个干员实例复用。
4. 不同干员即使共用同一个角色模板，也可以在 `.naga` 中拥有不同灵魂。
5. 新建干员时如果选择了某个角色模板，应当把该模板复制到该干员自己的 `IDENTITY.md`，而不是让实例运行时直接依赖 `characters/` 中的源文件。
